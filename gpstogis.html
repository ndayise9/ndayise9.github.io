From GPS to GIS

Getting GPS data into a GIS

    2 ways you can bring GPS coordinates into a GIS:
    1. Take GPS receiver into the field, write down the coordinates and why relevant info (attributes) you’re interested in, go back to the office and enter the coordinates/attributes into a text editor or spreadsheet software to create a table, import the file into the GIS. This is not the preferred method.
    2. Using a GPS receiver that includes a data logger, capture locations and attributes while in the field, transfer the GPS information to a computer, import to the GIS
    .

      Receivers that facilitate an automated approach will cost more but are worth it, and often necessary, if using GPS for GIS data collection.
      Receiver Examples

      • The GeoXM model will provide 1-3m accuracy and cost about $2000, the more sophisticated model GeoXH provides 30cm accuracy and costs around $6000. ESRI's Arcpad software can be loaded onto these devices, allowing you to create shapefiles during data collection.

            Mobile GPS
            • Some GPS receivers allow you to upload GIS data directly onto the unit, providing valuable reference information when out in the field. - Software like ESRI's ArcPad facilitates this.
              Preparation is the key What to keep in mind when undertaking GPS field reconnaissance:
            • Mission plan - check sateelite status and availability beforehand to head out at optimal time for collecting GPS data
            • Design a data dictionary before going out to collect data
            • Careful of how much data is loaded into receiver before going out
            • Assess the equipment before using it for project (is it an LCD screen which would make it hard to see
              A Data Dictionary
            • List all files in a database and the names and types of each field
            • Provide valuable system documentation which in turn makes it easier for data managers, data users, and application programmers to ultimately make use of the data
            • Implementing data standards into a data dictionary is how you succeed in data collection for an organization
                To create a data dictionary:
                1. Make a list of involved geographic features
                2. Identify attributes you wish to record for each feature
                3. Create the data dictionary
                4. Test your design
